Jesus Christ Facebook, Now You Decide to Fuck-Up Python Users?

Share

#“Jesus Christ Facebook, Now You Decide to Fuck-Up Python Users?”

Earlier in the week I got annoyed at Facebook for changing yet another API for no good reason (oh let's change blah_blah to blahBlah from now on, just for kicks). I've come to the realisation that they are breaking the web bit by bit. A site you built in 1997 willprobablybe 100% functional in 2011. A site you built in 2010 with deep Facebook integration, will probably be 90% broken in 2012 if you are notmaintainingit daily.

But this, this takes the fucking biscuit. Any site or service that uses their Python SDK including the millions of Django sites out there are now up shit creek as they just deleted it off Github!

Does anyone in Facebook understand version control? Did you not think to allow people to fork it first?

And why the hell would you need to save a couple of hundred kb of storage on GitHub? Just put a message in the goddammed README saying "no longer supported".

Facebook, you need togrowthe fuck up, fast. I have a 5 year old daughter who has more maturity and understands the impact of her decisions better than you.

The message from that developer says everything about what's broken in the Facebook organisation and mindset. Just because FB has a global team of developers working 24x7 on Facebook doesn't mean the rest of us do. If all we do is spend our days trying to fix things you break or change, we're not focused on our own businesses. So we can "simply" switch to urllib.urlopen can we? Will you pay a developer to do that for me?

I'm ok as I should have a GitHub clone of the repo somewhere I can use. Thank you Linus for creating a distributed version control system which prevents retarded behaviour like this from totally shafting people.
